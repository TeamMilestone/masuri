## 수정 완료

### 1. 체크섬 자릿수 누락 (3건) — 수정됨

`postprocess_code128`에서 Code Set C 확장 시 루프 종료 조건 버그.

**원인:** C에서 `for(i=1; i < dcode128->character-2; i++)`는 `character`가 `postprocess_c` 내부에서 변경되면 루프 종료 조건이 동적으로 반영됨. Rust에서는 `let len = dcode.code128.character`를 초기에 캡처하여 `while i < len.saturating_sub(2)`로 사용했기 때문에, `postprocess_c` 후 증가된 `character`가 반영되지 않아 마지막 문자가 잘림.

**수정:** `while i < (dcode.code128.character as usize).saturating_sub(2)` — 매 반복마다 현재 `character` 값을 읽도록 변경.

### 2. 단일 스레드 EAN lock 간섭 — 수정됨

단일 스레드 모드에서 EAN 디코더가 lock을 잡으면 Code128이 시작 검출을 포기하는 문제.

**수정:** `decode()` 함수의 기본 동작을 `scan_image_parallel()`로 변경. 병렬 모드에서는 행마다 독립 decoder를 사용하므로 lock 간섭 없음.
